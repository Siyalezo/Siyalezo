## 3. Reflections (STAR Method)

### Reflection on Coding in Markdown
- **Situation:** For the PRP370S assessment, I was required to create a digital portfolio showcasing my work-readiness training.
- **Task:** My objective was to use Markdown to code a professional CV and embed multimedia elements like my mock interview video.
- **Action:** I utilized GitHub Flavored Markdown (GFM) to structure my academic and professional history, ensuring all technical skills were highlighted using proper syntax and formatting.
- **Result:** I successfully created a structured, web-ready document that is easy to maintain and demonstrates my ability to use developer-standard documentation tools.

### Reflection on Mock Interview Video Experience
- **Situation:** Preparing for the transition into the professional ICT industry required a formal assessment of my interview skills.
- **Task:** I needed to record a mock interview that demonstrated my communication skills and technical knowledge to potential employers.
- **Action:** I prepared by reviewing my past projects, such as my work on the Clinic Management System, and practiced articulating my problem-solving process during my time as a student and part-time worker.
- **Result:** The recording process helped me identify my strengths in verbal communication and gave me a clear artefact to include in my portfolio as evidence of my professional growth.

### Reflection on the Use of GitHub Pages
- **Situation:** The final stage of the portfolio project was to make my work accessible to lecturers and external moderators via a live URL.
- **Task:** I had to deploy my repository using GitHub Pages to ensure the Markdown content rendered as a functional website.
- **Action:** I configured my repository settings to serve content from the main branch and verified that my video embed and Markdown CV were displaying correctly on the live site.
- **Result:** Deploying to GitHub Pages confirmed my ability to manage the full lifecycle of a basic web project, from coding to hosting.
